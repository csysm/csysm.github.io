git基本使用

3月 06, 2022 发布在 Git

git基本使用和推荐工作流

git项目创建

  1. 进入一个空目录，git bash here，使用git clone [url]下载代码
  2. 进入一个空目录，git bash here，使用git init创建项目

查看暂存区文件状态

查看所有文件状态git status

查看指定文件状态git status [fileName]

添加文件到暂存区

添加文件到暂存区git add [fileName]

提交文件到本地仓库

提交暂存区中的内容到本地仓库git commit -m "message"，参数-m代表提交信息

使用.gitignore忽略文件

1.在仓库中创建.gitignore文件，使用通配符配置忽略文件的类型

2.对于已经进行过git add操作的目录，如果新添加了.gitignore文件，则在进行git commit时，.gitignore文件不不会生效。需要清除暂存区缓存后再提交

提交文件到远程仓库

git remote add origin [url]在本地建立和远程仓库的关联

git remote rm origin删除和远程仓库的关联

git push origin master提交到远程仓库的master分枝

git pull --rebase origin master将本地仓库和远程仓库合并

生成公钥

ssh-keygen -t rsa -f "生成的文件名" -b "密钥长度,不写默认为2048" -C "注释,一般为用户名"

如果要为多个仓库配置不同密钥，生成多组密钥后需要修改~/.ssh/congif文件

这里为 github.com 网站指定的私钥放在文件 ~/.ssh/github中，对应的公钥文件 ~/.ssh/github.pub 注册到github平台 ssh key列表中，gitee网站也是一样处理，就可以了。
